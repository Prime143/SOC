📘 SOC INCIDENT RESPONSE PLAYBOOK

Welcome to the AEGIS-SOC Training Playbook. Follow this methodology to efficiently analyze, validate, and escalate cybersecurity threats.

PHASE 1: ANALYZE 🔍

Monitor the SIEM Log Feed: Attackers leave traces. Look for spikes in traffic, unusual ports, or critical events marked in RED (CRIT) or YELLOW (WARN).

  • Source IPs: Use the whois <IP> and geoip <IP> commands to determine if an IP is from an expected region or a known malicious ASN.
  • Ports: Use scan <IP> to check open ports. (e.g., Port 22 = SSH, Port 3389 = RDP).

PHASE 2: VALIDATE ✅

Not every alert is an attack. Validate if the alert is a False Positive (normal behavior) or True Positive (actual attack).

  • If an alert triggers for an internal IT admin doing normal work, you can classify it as a False Positive: classify ALRT-1234 FALSE_POSITIVE.
  • If a connection matches an attack signature (e.g. SQL Injection on port 80), confirm the severity (HIGH or CRITICAL).

PHASE 3: ESCALATE & CONTAIN 🚨

Once validated as malicious, you must block the threat actor to prevent data exfiltration or lateral movement.

  • Use the block <IP> command to add the attacker's external IP to the boundary firewall blocklist.
  • Warning: Never block internal IPs (e.g., 10.x.x.x, 192.168.x.x). Blocking internal infrastructure causes business outages!

💡 Pro-Tips for Beginners
